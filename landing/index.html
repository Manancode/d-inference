The Thesis

AI inference shouldn't
require trust

Cloud inference asks you to trust a corporation with your prompts. We're building the cryptographic alternative.

01

End-to-end encrypted

Prompts encrypted with the provider's X25519 public key before they leave your machine. NaCl box construction. The coordinator routes but never reads.

02

Hardware-attested

Every node proves identity through Apple Secure Enclave. P-256 keys in tamper-resistant silicon. Apple Enterprise Attestation CA chains verified.

03

Hardened runtime

Anti-debug, binary self-hash, SIP enforced, Hypervisor.framework memory isolation. The Mac owner cannot access inference memory.

04

Decentralized by design

No cloud lock-in. No GPU queues. A global mesh of Apple Silicon scored by decode speed, trust level, and hardware health in real time.

Architecture

How it works

Three steps. No GPU rentals. No cloud accounts. No data retention.

01

Send a request

Use the OpenAI-compatible API. Your prompt is encrypted before it leaves your machine. The coordinator never sees plaintext.

02

Routed to a verified node

The coordinator scores providers by decode speed, trust level, thermal state, and memory pressure — routes to the optimal machine.

03

Get your response

Inference runs in a hardened, isolated process. Results stream back encrypted. The node operator cannot see your data.

The question isn't whether AI inference will decentralize — it's whether the decentralized network can be trusted. That's the hard problem.
For Developers

One line to switch

Drop-in replacement for the OpenAI API. Change your base URL. Everything else stays the same.

python 
from openai import OpenAI

client = OpenAI(
    base_url="https://inference.eigencloud.xyz/v1",
    api_key="your-api-key"
)

response = client.chat.completions.create(
    model="qwen3-30b-a3b",
    messages=[{"role": "user", "content": "Hello!"}],
    stream=True
)

for chunk in response:
    print(chunk.choices[0].delta.content, end="")
